# Архитектурный отчет СУБД "Своя База"

## Примененный паттерн проектирования: Фасад (Facade)

### Где применен:
Паттерн реализован в классе `DatabaseFacade` (файлы `include/Database.h` и `src/Database.cpp`).

### Почему выбран данный паттерн:
Согласно MVP-требованиям (Пункт 2), все взаимодействия с СУБД должны происходить строго через API библиотеки без прямого доступа к структурам данных или файлам. Класс `DatabaseFacade` предоставляет единую, простую точку входа (`execute()`), скрывая под собой логику парсинга регулярных выражений, управление файловым потоком ввода-вывода таблиц и выполнение условий фильтрации. Это защищает внутреннюю целостность данных системы и упрощает интеграцию для конечного разработчика.
